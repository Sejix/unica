## Unica v0.5.1

Этот релиз закрывает большой набор Windows, runtime, RLM, metadata и release-инфраструктурных проблем после `v0.4.4`.

### Highlights

- Windows install/runtime path исправлен: packaged MCP запускает bundled `unica.exe` напрямую, без `bash`/shell wrappers.
- Добавлен Windows PowerShell 5.1 installer для установки release package без Git Bash/pwsh.
- `unica.meta.compile` теперь поддерживает все документированные metadata-типы, а не только ранний subset.
- RLM index lifecycle стал устойчивее: stale locks восстанавливаются автоматически, lock ownership защищает от гонок.
- `unica.code.search` стал native aggregator по backend sections: `bsl-analyzer`, `rlm`, `git grep`.
- Release pipeline получил BSP release assessment gate и публикуемые assessment reports.

### Fixed

- `unica.code.diagnostics` больше не передает неподдерживаемые параметры в `bsl-analyzer analyze` (#1).
- `unica.form.validate` теперь отклоняет bare `<Type>` values, которые ломают импорт через `ibcmd` (#2).
- `unica.cfe.borrow` покрывает enrichment уже заимствованного parent object и не перезаписывает существующий form module (#3, #4).
- `unica.subsystem.compile` сохраняет переданный UUID и генерирует уникальные UUID при отсутствии значения (#6).
- `unica.runtime.execute` больше не убивается внешним 120s timeout поверх `v8-runner` (#13).
- RLM-backed tools больше не зависают в вечном `rlm index building` после stale lock (#14).
- Windows RLM/code-search paths больше не запускают Unix `.sh` wrappers или устаревший `bsl-analyzer search --query` (#23, #25, #28).

### Dependencies and Tooling

- Обновлены bundled tool pins: `bsl-analyzer v0.2.48`, `v8-runner v0.5.1`, `rlm-tools-bsl/rlm-bsl-index v1.25.0`.
- Web test helpers получили storage operations.
- CI стал быстрее для обычных PR и строже для release/package-sensitive changes.
